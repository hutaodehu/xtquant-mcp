## Goal

把 fake / test 状态与正式实例证据隔离，恢复 `G0` 与 `G1` 的证据可信度。

## Implemented

1. 将 `tests/test_data_gateway_server.py` 改为使用仓库内独立 test runtime 路径，不再加载会写入 `instance/prod/*` 的 `data_gateway.example.yaml`，避免测试把 fake 结果写回 prod 证据目录。
2. 在 data/trade 资源读取层增加 prod 证据保护：当 `state_root` 落在 `instance/prod/state` 且 payload 声明 `source=fake` 时，资源读取返回 `fake_payload_blocked`，不再直接消费 fake payload。
3. 在 data/trade `/healthz` 补充证据来源字段：`evidence_scope`、`evidence_state_root`、`evidence_artifact_root`，便于 `G0/G1` 判断当前实例是否为 prod 证据上下文。
4. 在 data/trade call log 记录中补充 `evidence_scope` 与 `state_root`，降低后续排查时把测试日志误读为 prod 证据的风险。
5. 增加针对性单测覆盖：
   - data server 非 prod 证据范围健康字段验证；
   - data/trade 资源读取在 prod scope 下拦截 fake payload 的验证；
   - 现有 gateway server 单测回归通过。
6. 新增正式策略工件 `docs/OPS-001_ARTIFACT_DISPOSITION.md`，定义历史污染 artifact 的识别规则、分级、隔离/清理流程和失败分类口径，用于补齐 review 指出的“未标记污染处置缺口”。
7. 按 `OPS-001-review-202603300153.md` 回流修订策略工件，新增 mixed-content JSONL 日志处置约束与 manifest 最小完整性字段，避免“混合日志整文件按单一 fake 理由隔离”损伤证据完整性。

## Not Implemented
