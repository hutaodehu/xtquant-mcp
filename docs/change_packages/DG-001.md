## Goal

重构 `xtdata.status`，输出分层 readiness 与 resolved runtime endpoint。

## Implemented

1. `xtdata.status` 重构为分层 readiness 输出，包含 `import`、`connectivity`、`basic_query`、`metadata`、`job`、`lease` 六层状态。
2. 引入并固化 `configured_endpoint` 与 `resolved_runtime_endpoint` 双 endpoint 字段，同时保留原有 `xtdata_port` 兼容字段。
3. 调整总体 `ready` 计算为 core 层（`import` + `connectivity` + `basic_query`）；`metadata` 改为 non-blocking 层，避免 metadata 缺失误报整体不可用。
4. `xtdata.status` 在 metadata 缺失但 core 层正常时返回 `ok=True`、`ready=True`，并给出 `reason=metadata_missing` 与 warnings。
5. 同步更新 data gateway status 工具描述与 status 资源描述，使其明确体现 layered readiness + runtime endpoint 语义。
6. 保持现有测试文件不改动，本卡通过静态编译与内联 smoke 脚本验证 metadata 缺失场景不会把整体服务错误标记为不可用。

## Not Implemented

1. 未实现 `DG-002` 中的 `xtdata://leases/active` 深度 lease 健康建模（仅保留本卡所需 lease 基础状态层）。
2. 未在本卡引入新的 metadata 下载/修复能力面（保持现有能力边界）。
3. 未执行独立测试/审查阶段结论，本卡仅完成开发与自测记录。
