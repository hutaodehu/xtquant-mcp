## Goal

补齐 subscription lease 的 reconnect / rebuild recovery 语义，让数据侧订阅状态能表达“需要重建”和“为何需要重建”。

## Implemented

1. 在 `xtqmt_mcp/data_gateway/service.py` 中补齐 lease recovery 输出：
   - 单个 lease 现在同时暴露 `lease_state`、`needs_rebuild`、`rebuild_reason`、`recovery_action`、`reconnect_strategy`。
   - `recovery` / `health` 中补充 `callback_registered`、`callback_loop_alive`、`connection_alive`、`observed_event`、`created_age_seconds`、`last_event_age_seconds` 与 `resolved_runtime_endpoint`。
   - 聚合视图新增 `rebuild_reasons` 与 `recovery_summary`，让 agent 能按原因汇总 stale / stopped / needs_rebuild。
2. 保持输出与当前 runtime endpoint 模型对齐：
   - lease 相关输出继续同时包含 `configured_endpoint` 与 `resolved_runtime_endpoint`。
   - `xtdata.status` 的 `readiness.layers.lease` 增加 `active_count`、`rebuild_reasons`、`reconnect_strategy` 与 `proven_live_reconnect`。
3. 收紧 agent-facing 文案，避免高估 experimental subscription：
   - `xtdata://leases/active` 资源说明改为 rebuild-hint 视图，并明确“不是 durable reconnect 证明”。
   - `xtdata.subscribe.start/stop` 工具描述明确 experimental 与 “auto-rebuild 未被证明”。
   - `xtdata-service-recover` 等 prompt 改为引导读取 callback / connection liveness，并要求显式 rebuild。
4. 更新 `docs/MCP_DESIGN.md`：
   - 规范正文补充 `active` / `stale` / `stopped`、`needs_rebuild`、`rebuild_reason` 的目标契约。
   - Current Gaps 改为强调“恢复输出已存在，但 live reconnect / auto-rebuild 仍未被验证卡证明”。
5. 扩展 scoped tests，覆盖：
   - active lease
   - stale because no events
   - stale because xtdata connection lost
   - stale because callback loop missing
   - tool/resource/prompt 文案不把 experimental lease 误写成稳定能力

## Not Implemented

1. 没有实现 live reconnect 或自动 rebuild 执行路径；当前只输出恢复语义与显式 rebuild 指示。
2. 没有新增 live 环境验证，也没有把 experimental subscription 升格为稳定持久能力。
3. 没有修改与 subscription recovery 无关的 snapshot / history / calendar / instrument query 行为。

## Known Risks

1. 当前 stale / stopped 判定仍是本地 gateway 观测语义，不是 live market session 下的恢复证明；后续独立验证卡仍可能发现 xtdata 在真实断线/重连上的边缘情况。
2. `callback_loop_alive` 目前基于 gateway 内存中的 callback 注册与 seq 存在性推断，无法单靠这层输出证明底层推送线程一定健康。
3. `resolved_runtime_endpoint` 目前仍以当前配置 host/port + 连通性探测为主；若后续引入多 endpoint 解析或代理层，需要再次收口文档与输出。
