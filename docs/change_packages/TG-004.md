## Goal

把 Trade Gateway 的 session owner 与 trader 生命周期收口到服务端唯一管理模型。

## Implemented

1. 将 `GatewaySessionManager` 从“按 `account_id` 并存多 session”重构为“单 active owner session”模型：
   - 内部状态由 `_sessions` 字典改为 `_active_state` 单状态。
   - 增加 owner 生命周期字段：`owner_generation`、`owner_started_reason`。
2. 明确 owner 切换与重建时机：
   - 同 owner 且健康：复用 existing worker。
   - 同 owner 但健康失败：自动重建。
   - owner 不一致：默认拒绝并返回 `session_owner_conflict`，需 `force=true` 才允许切换 owner。
   - `session.close(account_id=...)` 对非 owner 账户返回 `session owner mismatch`，不误关当前 owner session。
3. 统一 `session.warm` / `session.status` / `session.close` 返回中的 owner 语义：
   - `status` 增加 `owner_account_id`、`owner_generation`、`owner_started_reason`。
   - 非 owner 查询返回 `session_owner_mismatch`，不再伪装为“该账户不存在”。
4. 将 `probe.connection` 收口到托管 session 执行路径：
   - 移除旁路临时 context 创建逻辑。
   - 改为通过 `GatewaySessionManager.execute(...)` 在当前 owner session 上执行探测。
   - 未 warm 或 owner 不匹配时返回明确错误（如 `session_not_ready`、`session_owner_mismatch`）。
5. 将 `snapshot.l1` 也切到托管 session 上执行，避免和 `session.status` 指向不同 trader 实例。
6. `trade://session/current` 资源读取改为读取 owner 当前状态（`status()`），而不是固定配置账户视角。
7. 更新 `trade://session/current` 资源描述，明确是 owner-managed session 摘要。
8. 修复 server 默认账户路由缺口（本次回退补丁）：
   - `session.status` / `session.close` 在无显式 `account_id` 时，不再回退到 `trade.account_id`，改为传空让 session manager 使用当前 active owner。
   - `probe.connection`、`snapshot.l1` 以及统一 `_session_manager.execute(...)` 的默认读写路径，不再强制传配置账户，默认走 active owner。
   - `fills.list` 若显式传 `account_id`，继续走 mismatch 诊断；若未显式传，执行时回落到当前 owner 上下文账户。
   - 对 execute 类工具补充 `RuntimeError` 到结构化错误 envelope 的映射，保留 `session_owner_mismatch` / `session_not_ready` 等诊断码。
9. 修复 `probe.connection` 的对外契约文案：
   - 将 tool description 从 “strict explicit-account” 改为 “active owner-managed session (no explicit account argument)”。
   - 保持实现与输入 schema 不变（无 `account_id` 入参，默认 active owner 路由）。

## Not Implemented
