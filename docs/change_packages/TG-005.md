## Goal

把 public `orders.list` 路径当前已被上游工件确认的 first blocker 收口为一个明确、可执行的 `G3` 设计缺陷修复任务。当前 blocker 必须保持 truthful：登录后复跑已经证明旧的 broker/login 环境阻断不再是当前真相，现阶段首要问题是 public `orders.list` 路径的 `fail_design`，错误文本为 `'NoneType' object has no attribute 'query_open_orders'`。

## Route Decision

- `upstream_truth`: [VAL-004.md](../task_cards/VAL-004.md), [VAL-004.md](../change_packages/VAL-004.md), [VAL-004-test-202603311831-post-login-rerun.md](../evidence_packs/VAL-004-test-202603311831-post-login-rerun.md), [VAL-004-review-202603311840.md](../reviews/VAL-004-review-202603311840.md)
- 开发边界：本卡只处理 public `orders.list` 在 `G3` public read 路径上的设计/实现缺陷收口，不处理 `order.place`，不启动 [VAL-003.md](../task_cards/VAL-003.md)，不宣称任何 `G4 ready` 或写路径放行。
- 设计目标：统一 warm-health `orders.list` 与 public `orders.list` 的对象生命周期和契约表达，避免继续出现“warm health 成功但 public 调用直接因空对象崩溃”的分叉叙事。
- 分类约束：若修复后仍失败，后续工件必须能把结果继续分清为 `fail_design`、`fail_env` 或显式 degraded contract，不能再把当前 `NoneType.query_open_orders` 回退混写成笼统 `server_env_not_ready`。

## Implemented

1. 已在 [xtqmt_mcp/trade_gateway/bootstrap.py](../../xtqmt_mcp/trade_gateway/bootstrap.py) 对齐 runtime wiring：
   - `session.warm` 保持 `need_shadow=True`，但不再要求 `need_broker=True`；warm/status 继续只承担 owner-managed warm health 的 shadow-only 读职责。
   - public `orders.list` 继续声明 `need_shadow=True` 和 `need_broker=True`，使其 runtime 依赖与现有 public fallback contract 一致，而不是只装 broker 却让 fallback 永远失去对象前提。
2. 已在 [xtqmt_mcp/trade_ops.py](../../xtqmt_mcp/trade_ops.py) 收口 public `orders.list` 的 non-crash contract：
   - `self.broker is None` 时不再触发 `NoneType.query_open_orders`；优先尝试显式 public shadow fallback。
   - 当 broker adapter 缺席但 active owner-managed shadow 可复用时，返回 `ok=true`、`source=active_owner_shadow`、`read_scope=public_fallback`、`degraded=true`、`fallback_reason=broker_missing` 的 machine-readable degraded 结果。
   - 当 broker connect 失败且 shadow 可复用时，继续返回显式 degraded fallback，并保持 `fallback_reason=broker_connect_failed`。
   - 当 broker 缺席且 shadow 不可用时，返回 `ok=false`、`code=orders_list_broker_missing`、`failure_classification=fail_design` 的 machine-readable 失败结果，不再让 server 兜底成 `server_env_not_ready`。
   - 当 broker 查询抛出 connect failure 且 shadow 不可复用时，返回 `code=connect_failed` 与 `failure_classification=fail_env`；其他 broker 读取异常则返回 `code=orders_list_broker_read_failed` 与 `failure_classification=fail_design`。
3. 已在 [xtqmt_mcp/trade_gateway/server.py](../../xtqmt_mcp/trade_gateway/server.py) 校正文案，明确 public `orders.list` 的 shadow fallback 是 public degraded 路径，且 warm-health-only shadow 结果不等于 public broker readiness。
4. 已补 deterministic tests：
   - [tests/test_trade_ops_warm_health.py](../../tests/test_trade_ops_warm_health.py) 新增 broker 缺席 fallback 与 broker+shadow 同时缺席时的 machine-readable fail-design 用例。
   - [tests/test_trade_gateway_server.py](../../tests/test_trade_gateway_server.py) 新增 wiring consistency 断言，并锁定 `orders.list` tool definition 中对 warm-health/public 边界的说明。
