## Goal

把 `session/account` 在 schema、资源、会话和执行路径上的表达统一为单账户优先主契约，消除“服务端上下文”和“每次调用输入”混写。

## Scope In

1. 统一工具 schema 中账户维度的表达方式。
2. 统一默认账户上下文、显式账户输入和资源输出的口径。
3. 为未来多账户扩展保留清晰扩展位，但不在当前正文中混入多账户主流程。

## Scope Out

1. 不实现正式多账户支持。
2. 不改写与账户维度无关的数据侧能力。

## Done Means

agent 能明确判断账户维度是服务端主上下文还是显式输入；Trade Gateway 的 schema、resource 和执行路径对账户的表达不再互相打架。
