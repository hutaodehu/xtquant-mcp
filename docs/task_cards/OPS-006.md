## Goal

把通用交付技能套件的核心 contract 从“可讨论的设计草案”收口为“可被 review 的正式 schema 与 authority 边界”，为后续 adapter 和实现卡提供稳定上游。

## Scope In

1. 收口 `route_decision`、`risk_profile`、`acceptance_contract`、`gate_result` 的最小字段、字段关系和 fail-closed 规则。
2. 收口 `workflow-router`、`acceptance-analysis`、`mode-*`、`evidence-gate` 之间的职责边界和 authority vocabulary。
3. 补齐或收敛与 agent/tool routing、多 agent handoff、docs-only / shared-change 场景对应的 golden examples。

## Scope Out

1. 不修改 `.agents/skills/*` 的实际执行逻辑、prompt、tool 选择或权限行为。
2. 不在本卡内定义当前仓库专用 `TaskCard/ChangePack` adapter 映射。
3. 不进行插件化、跨仓库发布或分发形态决策翻案。

## Done Means

后续执行者不再需要临时发明通用交付 suite 的核心 contract，至少可以稳定回答：

1. `workflow-router` 应输出什么。
2. `acceptance-analysis` 应把哪些 gate、authority、evidence freshness 固化进 contract。
3. 哪些场景必须停在 analyze-only，哪些场景至少升格到 gated-change。

## Notes

若后续需要真实修改 `.agents/skills/*` 的行为，应从本卡继续拆实现卡，而不是把 design contract 和运行逻辑混做一张卡。
