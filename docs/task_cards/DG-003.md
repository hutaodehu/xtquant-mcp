## Goal

把 subscription lease 从“当前可读状态”补齐到“带 reconnect / rebuild 恢复语义的模型”，让 agent 能判断租约是否陈旧、是否需要重建以及为什么需要重建。

## Scope In

1. 补齐 lease 的恢复状态表达，包括 callback loop、连接存活、`needs_rebuild`、`rebuild_reason` 和相关状态转移语义。
2. 明确 `stopped`、`stale`、`needs_rebuild` 等状态边界，并保持与当前 runtime endpoint 对齐。
3. 同步资源、工具输出和 prompt 文案，避免 agent 把 experimental lease 误读成稳定持久订阅。

## Scope Out

1. 不在本卡内宣称 live reconnect / auto-rebuild 已在真实环境证明。
2. 不把 experimental subscription 直接升格为稳定长期能力。
3. 不修改与 subscription recovery 无关的数据查询能力。

## Done Means

`xtdata://leases/active` 及相关输出可以清晰表达 recovery 语义；agent 能区分“当前租约可读”“当前租约需要重建”“当前恢复能力尚未被 live 证明”这几种不同状态。

## Notes

若后续要证明 live reconnect / auto-rebuild 已成立，应以新的验证卡进入 `dev -> test -> review`，不得复用本卡结论直接宣称通过。
