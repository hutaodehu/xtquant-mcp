## Goal

把 Trade Gateway 的会话管理收口为服务端唯一 owner 模型，避免多条 trader 路径共享或争用同一 `session_id`。

## Scope In

1. 统一 `session.warm`、`session.status`、`session.close` 背后的 session owner。
2. 明确单 trader 生命周期、重建时机与健康判定。
3. 让资源、状态和审计都指向同一个服务端托管 session。

## Scope Out

1. 不引入多账户并发会话模型。
2. 不在本卡内完成真实下单验收。

## Done Means

Trade Gateway 不再存在“看起来是一个 session，实际上由多条 trader 路径共享”的歧义；`G3` 只读链路可以基于同一 owner 模型完成预热、状态查询和只读探测。
