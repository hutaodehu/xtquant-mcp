## Goal

把订阅能力建模为 experimental subscription lease，并在 `xtdata://leases/active` 中输出租约健康信息，而不是只保留 `subscription_id`。

## Scope In

1. 输出 callback loop 是否存活、连接是否仍存活、最近事件时间、是否需要重建租约、停止原因。
2. 让租约状态能关联当前 resolved runtime endpoint。
3. 明确 subscription capability 仍是 experimental，而不是已证稳定能力。

## Scope Out

1. 不承诺自动重连已经被真实环境验证。
2. 不把 subscription lease 升格为长期稳定持久句柄。

## Done Means

agent 读取 `xtdata://leases/active` 时，能够区分“租约对象仍存在”和“租约健康仍成立”；租约视图不再误导 agent 把状态残留当成活跃订阅。

## Notes

本卡的主要价值是修正 agent 心智模型，而不是扩大订阅能力面。
