## Goal

把 read-only preflight 与 write-permission preflight 从总体 readiness 中拆开，让只读诊断不再被写权限前置条件错误阻断。

## Scope In

1. 拆分连接、登录、只读查询和写权限的 readiness 表达。
2. 将 `up_queue_xtquant` 等写权限前置条件限制在写路径相关状态。
3. 更新健康输出、诊断输出和错误分类，让 agent 能看见两层能力边界。

## Scope Out

1. 不在本卡内完成真实下单执行。
2. 不把 prompt 当作唯一安全边界替代服务端 gate。

## Done Means

缺失写权限时，Trade Gateway 仍能完成 `G3` 所需的只读探测并返回准确失败分类；真实写路径则继续被明确阻断。

## Notes

本卡完成后，`VAL-002` 才能验证“只读成立但写权限缺失”的正确分层行为。
