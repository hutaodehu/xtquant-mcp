## Goal

重构 `xtdata.status`，让它输出分层 readiness 和 resolved runtime endpoint，而不是把静态配置值或元数据能力混成单一 `ready`。

## Scope In

1. 分离 import、connectivity、basic query、metadata、job、lease 六层状态。
2. 同时输出 configured endpoint 与 resolved runtime endpoint。
3. 让 `xtdata.status` 和相关资源面对 agent 暴露运行时真相，而不是示例值。

## Scope Out

1. 不在本卡内完成 subscription lease 深度健康建模。
2. 不把 sector metadata 预下载工具扩展成新的正式能力面。

## Done Means

`xtdata.status` 不再因 sector metadata 缺失而误报整体不可用；agent 能明确看到当前实际解析到的 host/port 与各层 readiness。

## Notes

本卡是 Data Lane 的基础卡；`DG-002` 和 `VAL-001` 都依赖本卡的新状态模型。
