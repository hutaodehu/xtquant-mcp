## Goal

把登录后复跑暴露出来的 public `orders.list` 首要阻断收口为明确的实现修复任务，避免继续把当前问题混写成 broker/login 环境阻断。

## Scope In

1. 修复 public `orders.list` 路径上的空对象调用缺陷，当前错误文本为 `'NoneType' object has no attribute 'query_open_orders'`。
2. 对齐 warm-health `orders.list` 与 public `orders.list` 的契约边界，避免出现“warm health 成功但 public 调用立即崩溃”的分叉叙事。
3. 保持错误分类 truthful：若修复后仍失败，必须能明确区分 `fail_design`、`fail_env` 或 degraded contract，而不是回退成笼统 `server_env_not_ready`。
4. 为新的独立 `G3` rerun 创造条件，但不在本卡内直接做 live 放行结论。

## Scope Out

1. 不执行 `order.place`，不启动 [VAL-003.md](./VAL-003.md)。
2. 不扩展新的交易策略能力，也不引入新的多账户或多 session 模型。
3. 不把只读/probe 成功外推为 `G4 ready`；本卡编写时引用的 `probe.connection.write_permission_ready=true` 属于当时的字段语义，当前 contract 读取应改以 [MCP_DESIGN.md](../MCP_DESIGN.md) 和 [OPERATIONS_RUNBOOK.md](../OPERATIONS_RUNBOOK.md) 中的 `write_permission_precheck_ok`、`write_session_alignment.same_plan_verdict`、`write_permission_ready` / `write_authority_ready` 为准。

## Done Means
