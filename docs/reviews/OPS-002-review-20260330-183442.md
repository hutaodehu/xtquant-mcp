## Severity

## Impact

OPS-002 的目标是把 live 运维和 blocker recovery 固化成正式执行入口，因此 runbook 里的 operator-facing 输出规则本身就是 contract 的一部分。当前把 board 级 `needs_fix` 混入 formal review verdict，会带来两个直接风险：

1. 后续 review agent 可能在 `ReviewPack` 中写出非标准 verdict，违反仓库统一词汇约束。
2. 主控或 reconcile 流会同时面对 formal artifact verdict 与 board field enum 两套口径，增加误读和错误同步风险。

除这一点外，本次审查范围内未发现新的 `G3 pass -> G4 ready` 误导，也未发现把 `broker_blocked` 写成已解除；runbook 对 `VAL-002` accepted posture、`VAL-003` 阻断状态、入口 backlinks 和 artifact chain 基本保持一致。

## Required Fix

1. 把 [OPERATIONS_RUNBOOK.md](../OPERATIONS_RUNBOOK.md) `Operator Output Rules` 中的 review verdict 规则改成仅使用正式 verdict vocabulary，不再出现 `needs_fix`。
2. 若需要保留 `needs_fix`，必须明确它只属于 board / `Review Result` 字段，不属于 `ReviewPack` 或其他 formal artifact 的结论词汇。
3. 修正后重新走独立测试或最小文档复核，确认 runbook 内不再混淆 formal verdict 与 board enum。
