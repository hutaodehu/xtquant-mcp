2. dev 修复保持了 truthful classification，没有把已确认的设计缺陷重新混写成 `server_env_not_ready`，也没有把 warm-health 成功偷换成 public success。

   - 同一实现里，只有 broker connect failure 且 shadow 不可复用时，才返回 `connect_failed` + `failure_classification=fail_env`；这与“设计问题”和“环境问题”分层一致。
   - [xtqmt_mcp/trade_gateway/bootstrap.py](../../xtqmt_mcp/trade_gateway/bootstrap.py) 已把 `session.warm` 与 public `orders.list` 统一为同时需要 broker + shadow 的 runtime wiring；[xtqmt_mcp/trade_gateway/server.py](../../xtqmt_mcp/trade_gateway/server.py) 也显式声明 public `orders.list` 的 shadow fallback 是 public degraded path，且 “Warm-health-only shadow results are not equivalent to public broker readiness.”
   - 配套测试 [tests/test_trade_ops_warm_health.py](../../tests/test_trade_ops_warm_health.py) 与 [tests/test_trade_gateway_server.py](../../tests/test_trade_gateway_server.py) 明确锁定了三种分支：broker success、public degraded fallback、以及 broker+shadow 同时缺席时的 machine-readable `fail_design`，因此当前实现没有再把 warm health 结果冒充 public 成功。
