# v2.0.0-alpha.1 发布说明

## 定位

`v2.0.0-alpha.1` 是面向开源前冻结的脱敏候选版本。它保留可运行的 `xtqmtTradeGateway` / `xtqmtDataGateway` 样例项目、配置模板、测试与运行手册，不包含真实账户、真实成交、真实持仓、真实运行态或本机私有路径。

## 功能更新

1. 运行时端口与 readiness 语义改为动态解析，`58610` 不再作为硬编码协议常量。
2. MiniQMT 登录窗口识别增强，避免把主交易窗口误判为登录窗口。
3. 行情 `L1` 快照保留五档盘口，并支持 `get_full_tick` 优先路径。
4. Data Gateway 增加现代 MCP 工具面：`gateway.health`、`calendar.resolve_trade_day`、`integrity.plan`、`sector.*`、`market.*`、`bulk.sync_job.*`、`artifact.manifest`、`qlib.*`。
5. Data Gateway 增加 qlib 同步运行时、完整性计划和 acceptance 检查。
6. Trade Gateway 增强 session warm timeout 状态保持、owner-managed broker reuse、订单状态与成交真相查询。
7. 公开候选保留 packet readiness helper、wake 脚本和 flow-smoke 脚本；内部 controller-direct live harness 已移出公开 baseline。

## 文档更新

1. `README.md`、`docs/OPERATIONS_RUNBOOK.md`、`docs/MCP_DESIGN.md`、`docs/CURRENT_STATUS.md`、`docs/ACCEPTANCE_STANDARD.md` 已更新为公开运行口径。
2. `docs/env_snapshots/`、`docs/evidence_packs/`、`docs/reviews/` 和 `docs/superpowers/` 只保留 README stub。
3. `docs/release/*` 记录开源整理、归档、冻结和验证结论。

## 发布限制

当前整理分支仍不能直接快进公开 `main`：远端基线历史已命中敏感模式。真正开源发布必须使用脱敏公开历史或新公开仓，并在用户再次确认后执行推送、tag 和 release。
